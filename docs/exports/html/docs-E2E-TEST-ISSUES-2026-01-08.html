Issue #1: Validation Error Display Not Showing

Status: FIXED Tests Affected: 5 (all browsers) Category: Authentication Severity: High

Description: Login validation errors were not appearing in the UI when users entered invalid credentials. Tests expected error messages matching /invalid|incorrect|failed/i but received "User not found" or no error at all.

Playwright Strict Mode **Issue:** Locators must resolve to exactly one element **Solution:** Use `.first()`, `.last()`, or `.nth()` for multiple matches 
    // ❌ Fails with strict mode violation if multiple matches
const errorMessage = page.getByText(/invalid/i);

// ✅ Explicitly handles multiple matches
const errorMessage = page.getByText(/invalid/i).first();
    ### 2. Content-Based vs Visibility-Based Assertions **Issue:** `.isVisible()` unreliable for conditional rendering **Solution:** Check text content directly 
    // ❌ Less reliable
const hasEmptyState = await emptyState.isVisible();

// ✅ More reliable
const pageContent = await page.locator('body').textContent();
const hasEmptyState = pageContent?.toLowerCase().includes('no scripts');
